Placeholder Nodes

What is a placeholder node?

Placeholder nodes are automatically created when importing BloodHound data. They represent Active Directory objects that are referenced by relationships but were not found in the imported data files.

Why does this happen?

  • The node may be defined in a different BloodHound data file that wasn't imported
  • The node may have been filtered out during BloodHound collection
  • The object may no longer exist in Active Directory but is still referenced by other objects

What should I do?

You can import additional BloodHound data files to populate these placeholder nodes with their actual properties. If you have the complete data set, the placeholder nodes will be updated with real data upon re-import.
